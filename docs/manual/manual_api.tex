\section{Introducción}
Este documento es una guía práctica para usar el Sistema de Torneos y Apuestas \textbf{TournaBet}.
El sistema permite gestionar torneos deportivos donde los participantes realizan selecciones
y acumulan puntos según los resultados.

\subsection{¿Qué es TournaBet?}
Es una plataforma para crear torneos de pronósticos deportivos (como caballos, fútbol, beisbol, etc.)
donde:
\begin{itemize}
    \item \textbf{Admin}: Crea torneos, categorías, sesiones, eventos globales y los asigna a torneos
    \item \textbf{Usuario}: Se registra, participa, hace pronósticos y acumula puntos en el leaderboard
\end{itemize}

\subsection{Novedades de la Versión 2.0}
Esta versión introduce importantes mejoras:
\begin{itemize}
    \item \textbf{Eventos Globales}: Los eventos se crean una vez y pueden reutilizarse en múltiples torneos
    \item \textbf{Tabla TournamentEvents}: Nueva tabla intermedia para vincular eventos a torneos
    \item \textbf{Puntos por Tipo de Selección}: Configuración personalizada de puntos ganados por cada tipo de a
\item \textbf{SuperLine}: Línea especial para sesiones de playoffs o finales
    \item \textbf{Catálogo Global de Competidores}: Competidores pueden tener odds, RL y SRL específicos por evento
\end{itemize}

\subsection{Usuario Administrador Inicial}
Al ejecutar la aplicación por primera vez, se crea automáticamente:
\begin{itemize}
    \item \textbf{Email}: admin@betsystem.com
    \item \textbf{Password}: Admin123!
\end{itemize}

\subsection{Tecnologías Utilizadas}
\begin{itemize}
    \item \textbf{Backend}: Go 1.21+ con framework Gin Gonic, ORM GORM
    \item \textbf{Frontend}: React 18+ con Vite, TypeScript, Redux Toolkit, Tailwind CSS v4
    \item \textbf{Base de Datos}: MySQL o PostgreSQL (configurable via GORM)
    \item \textbf{Autenticación}: JWT (JSON Web Tokens)
\end{itemize}

\subsection{Conceptos Clave del Sistema}
\begin{enumerate}
    \item \textbf{Eventos Globales}: Los eventos (partidos, carreras) se crean una vez y pueden reutilizarse en múltiples torneos mediante la tabla \texttt{TournamentEvent}
    
    \item \textbf{Tabla TournamentEvent}: Relación N:N entre Tournament y Event. Permite asignar un evento global a una sesión específica de un torneo
    
    \item \textbf{Sesiones}: Cada torneo tiene múltiples sesiones (jornadas). Los eventos se asignan a sesiones específicas
    
    \item \textbf{Competidores por Evento}: Cada evento tiene competidores con odds, RL, SRL específicos. El campo \texttt{IsFavorite} determina Macho (true) o Hembra (false)
    
    \item \textbf{Tipos de Selección}: macho, hembra, alta, baja, runline, sralta, srbaja
    
    \item \textbf{Puntuación}: Los puntos por tipo de selección se configuran en \texttt{TournamentSettings.PointsBySelectionType}
\end{enumerate}

\subsection{Estado Actual del Proyecto}
El proyecto está en \textbf{modo de desarrollo}. La base de datos puede eliminarse y recrearse según sea necesario para modificar los modelos.

    \item \textbf{Frontend}: Las páginas principales están en \texttt{src/pages/admin/} y \texttt{src/pages/user/}
    \item \textbf{API Types}: Tipos TypeScript en \texttt{bets-frontend/src/types/api.ts}
    \item \textbf{Configuración}: El backend usa variables de entorno (crear \texttt{.env} con DB_HOST, DB_PORT, DB_USER, DB_PASSWORD, DB_NAME, JWT_SECRET)
\end{itemize}

\section{Flujo de Uso - Paso a Paso}

\subsection{Paso 1: Crear una Categoría}
Las categorías organizan los torneos (Fútbol, Caballos, Beisbol, etc.):

\begin{lstlisting}[language=bash]
POST /api/v1/admin/categories
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "name": "Fútbol",
  "description": "Torneos de fútbol soccer",
  "icon": "soccer-ball"
}
\end{lstlisting}

\subsection{Paso 2: Crear un Torneo}
El admin crea un torneo con configuración de puntos:

\begin{lstlisting}[language=bash]
POST /api/v1/admin/tournaments
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "name": "Copa Mundial 2026",
  "category_id": 1,
  "description": "Pronósticos del mundial",
  "start_date": "2026-06-01T00:00:00Z",
  "end_date": "2026-07-15T23:59:59Z",
  "entry_fee": 500,
  "max_participants": 100,
  "settings": {
    "selections_per_session": 3,
    "total_sessions": 8,
    "required_selection_types": ["macho", "alta", "baja"],
    "points_per_selection_type": {
      "macho": 10,
      "hembra": 15,
      "alta": 8,
      "baja": 8,
      "runline": 12,
      "sralta": 20,
      "srbaja": 20
    }
  }
}
\end{lstlisting}

\textbf{Campos importantes:}
\begin{itemize}
    \item \textbf{max\_participants}: Límite máximo de usuarios que pueden inscribirse (0 = ilimitado)
    \item \textbf{points\_per\_selection\_type}: Puntos awarded por cada tipo de selección acertada
    \item \textbf{required\_selection\_types}: Tipos de selecciones obligatorias por sesión
\end{itemize}

\subsection{Paso 3: Crear Sesiones (Jornadas)}
Cada torneo tiene varias sesiones. Ej: 8 jornadas para un mundial:

\begin{lstlisting}[language=bash]
POST /api/v1/admin/sessions
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "tournament_id": 1,
  "session_number": 1,
  "start_time": "2026-06-01T10:00:00Z",
  "end_time": "2026-06-01T14:00:00Z",
  "super_line": null,
  "description": "Jornada 1 - Fase de grupos"
}
\end{lstlisting}

\textbf{Campo especial - SuperLine:}
\begin{itemize}
    \item Para sesiones normales de playoffs o finales, usar \textbf{super\_line} para establecer una línea especial
    \item Los tipos \textbf{sralta} y \textbf{srbaja} usarán esta línea en lugar de la línea regular
\end{itemize}

\subsection{Paso 4: Crear Eventos Globales (NUEVO)}
Los eventos se crean una vez y pueden reutilizarse en múltiples torneos:

\begin{lstlisting}[language=bash]
POST /api/v1/admin/events
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "name": "España vs Francia",
  "venue": "Santiago Bernabéu",
  "line": 2.5,
  "total_score": 0,
  "start_time": "2026-06-15T20:00:00Z",
  "status": "scheduled"
}
\end{lstlisting}

\textbf{Campos del Evento:}
\begin{itemize}
    \item \textbf{name}: Nombre del evento (ej: "España vs Francia", "Carrera 5")
    \item \textbf{venue}: Lugar del evento (estadio, hipódromo, etc.)
    \item \textbf{line}: Línea base para apuestas Alta/Baja
    \item \textbf{total\_score}: Score total actual (se actualiza al liquidar)
    \item \textbf{start\_time}: Hora de inicio del evento
    \item \textbf{status}: Estado (scheduled, live, settled, cancelled)
\end{itemize}

\subsection{Paso 5: Asignar Evento a un Torneo}
Vincular un evento global a un torneo específico:

\begin{lstlisting}[language=bash]
POST /api/v1/admin/tournament-events
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "event_id": 1,
  "tournament_id": 1,
  "session_id": 1,
  "order": 1
}
\end{lstlisting}

\textbf{O alternativamente, ver eventos disponibles para asignar:}
\begin{lstlisting}[language=bash]
GET /api/v1/admin/events/available?tournament_id=1
Authorization: Bearer <token_admin>
\end{lstlisting}

\subsection{Paso 6: Establecer Competidores}
Para cada evento, definir los competidores (equipos, caballos, etc.):

\begin{lstlisting}[language=bash]
POST /api/v1/admin/events/{id}/competitors
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "competitors": [
    {
      "name": "Selección Española",
      "assigned_number": 1,
      "odds": 1.85,
      "rl": -1.5,
      "srl": -2.5
    },
    {
      "name": "Selección Francesa", 
      "assigned_number": 2,
      "odds": 2.10,
      "rl": 1.5,
      "srl": 2.5
    }
  ]
}
\end{lstlisting}

\textbf{Campos del Competidor por Evento:}
\begin{itemize}
    \item \textbf{name}: Nombre del competidor
    \item \textbf{assigned\_number}: Número asignado (1 para local, 2 para visitante)
    \item \textbf{odds}: Cuota decimal para apuesta de línea de dinero
    \item \textbf{rl}: Runline (para beisbol)
    \item \textbf{srl}: Super Runline (para sesiones con SuperLine)
\end{itemize}

\subsection{Paso 7: Crear Selecciones (Opciones de Apuesta)}
Las selecciones son las opciones en las que los usuarios pueden apostar:

\begin{lstlisting}[language=bash]
POST /api/v1/admin/events/selections
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "event_id": 1,
  "description": "España gana",
  "selection_type": "macho",
  "competitor_id": 1,
  "line": null,
  "points_for_win": 10
}
\end{lstlisting}

\begin{lstlisting}[language=bash]
POST /api/v1/admin/events/selections
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "event_id": 1,
  "description": "Más de 2.5 goles",
  "selection_type": "alta",
  "competitor_id": null,
  "line": 2.5,
  "points_for_win": 8
}
\end{lstlisting}

\begin{lstlisting}[language=bash]
POST /api/v1/admin/events/selections
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "event_id": 1,
  "description": "Menos de 2.5 goles",
  "selection_type": "baja",
  "competitor_id": null,
  "line": 2.5,
  "points_for_win": 8
}
\end{lstlisting}

\textbf{Reglas:}
\begin{itemize}
    \item Para tipos \textbf{macho} y \textbf{hembra}: Se debe especificar \textbf{competitor\_id}
    \item Para tipos \textbf{alta}, \textbf{baja}, \textbf{sralta}, \textbf{srbaja}: Se debe especificar \textbf{line}
    \item \textbf{points\_for\_win}: Puntos awarded por esta selección (sobrescribe la configuración del torneo)
\end{itemize}

\subsection{Paso 8: Usuario se Registra e Inscribe}
El usuario se registra, verifica su email, recarga su billetera y se inscribe:

\begin{lstlisting}[language=bash]
# Registro
POST /api/v1/auth/register
Content-Type: application/json

{
  "email": "usuario@email.com",
  "password": "Password123!",
  "name": "Juan Pérez",
  "username": "juanperez"
}
\end{lstlisting}

\begin{lstlisting}[language=bash]
# Login
POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "usuario@email.com",
  "password": "Password123!"
}
\end{lstlisting}

\begin{lstlisting}[language=bash]
# Recargar billetera
POST /api/v1/wallet/deposit
Content-Type: application/json
Authorization: Bearer <token_usuario>

{
  "amount": 1000,
  "payment_method_id": 1
}
\end{lstlisting}

\begin{lstlisting}[language=bash]
# Inscribirse al torneo
POST /api/v1/tournaments/1/join
Authorization: Bearer <token_usuario>
\end{lstlisting}

\subsection{Paso 9: Usuario hace Pronósticos}
El usuario hace sus selecciones para una sesión específica:

\begin{lstlisting}[language=bash]
POST /api/v1/tournaments/1/sessions/picks
Content-Type: application/json
Authorization: Bearer <token_usuario>

{
  "session_id": 1,
  "selection_ids": [1, 2, 3]
}
\end{lstlisting}

\textbf{Validaciones:}
\begin{itemize}
    \item El número de selecciones debe coincidir con \textbf{selections\_per\_session} del torneo
    \item Los tipos de selecciones deben incluir los \textbf{required\_selection\_types}
    \item Las selecciones deben corresponder a eventos de la sesión especificada
    \item El usuario no puede hacer pronósticos después del \textbf{end\_time} de la sesión
\end{itemize}

\subsection{Paso 10: Liquidar el Evento}
Cuando termina el evento, el admin establece los resultados:

\begin{lstlisting}[language=bash]
POST /api/v1/admin/events/{id}/settle
Content-Type: application/json
Authorization: Bearer <token_admin>

{
  "results": [
    {"competitor_id": 1, "final_score": 2, "position": null},
    {"competitor_id": 2, "final_score": 1, "position": null}
  ]
}
\end{lstlisting}

\textbf{Para carreras de caballos:}
\begin{lstlisting}[language=c]
{
  "results": [
    {"competitor_id": 1, "final_score": null, "position": 1},
    {"competitor_id": 2, "final_score": null, "position": 2},
    {"competitor_id": 3, "final_score": null, "position": 3},
    {"competitor_id": 4, "final_score": null, "position": 4}
  ]
}
\end{lstlisting}

\textbf{El sistema automáticamente:}
\begin{enumerate}
    \item Actualiza el \textbf{total\_score} del evento
    \item Evalúa cada selección (alta/baja/macho/hembra/etc.)
    \item Marca las selecciones ganadoras/perdedoras
    \item Suma los puntos configurados a los usuarios que acertaron
    \item Actualiza el leaderboard del torneo
\end{enumerate}

\subsection{Paso 11: Consultar Leaderboard}
Ver el ranking de participantes:

\begin{lstlisting}[language=bash]
GET /api/v1/tournaments/1/leaderboard
\end{lstlisting}

\begin{lstlisting}[language=bash]
GET /api/v1/tournaments/1/leaderboard?session_id=1
\end{lstlisting}

\end{lstlisting}
(Repetir para las 5 carreras)

\subsection{6. Establecer Competidores para cada Carrera}
\begin{lstlisting}[language=c]
{
  "competitors": [
    {"name": "El Trueno", "assigned_number": 1, "odds": 2.5},
    {"name": "La Flecha", "assigned_number": 2, "odds": 3.0},
    {"name": "Velocista", "assigned_number": 3, "odds": 4.0},
    {"name": "Estrellita", "assigned_number": 4, "odds": 5.0}
  ]
}
\end{lstlisting}

\subsection{7. Crear Selecciones para cada Evento}
Para cada carrera, crear las 4 selecciones requeridas:
\begin{itemize}
    \item Macho (apostar a un caballo específico)
    \item Hembra (apostar a otro caballo)
    \item Alta (tiempo total mayor a línea)
    \item Baja (tiempo total menor a línea)
\end{itemize}

\subsection{8. Liquidar Resultados}
Cuando termina la carrera:
\begin{lstlisting}[language=c]
{
  "results": [
    {"competitor_id": 1, "position": 1, "final_time": 120.5},
    {"competitor_id": 2, "position": 2, "final_time": 121.0},
    {"competitor_id": 3, "position": 3, "final_time": 122.0},
    {"competitor_id": 4, "position": 4, "final_time": 123.0}
  ]
}
\end{lstlisting}

El sistema determina automáticamente qué selecciones fueron ganadoras según:
\begin{itemize}
    \item \textbf{Macho}: El competidor con position=1 fue el seleccionado
    \item \textbf{Hembra}: El resto de competidores
    \item \textbf{Alta/Baja}: Según el tiempo total vs la línea
\end{itemize}

\subsection{Rate Limiting}
El sistema implementa rate limiting:
\begin{itemize}
    \item \textbf{Autenticación}: 5 intentos por minuto
    \item \textbf{API general}: 60 requests por minuto
    \item \textbf{Escritura} (POST/PUT/DELETE)**: 30 requests por minuto
\end{itemize}

\subsection{Ejemplo: Integración Frontend - Obtener Torneos}
\begin{lstlisting}[language=c]
// Obtener lista de torneos
const response = await fetch('http://localhost:8080/api/v1/tournaments');
const data = await response.c();

if (data.success) {
  const tournaments = data.data;
  tournaments.forEach(t => {
    console.log(`${t.name} - ${t.category.name}`);
  });
}
\end{lstlisting}

\subsection{Ejemplo: Integración Frontend - Hacer Picks}
\begin{lstlisting}[language=c]
// Autenticación
const loginRes = await fetch('http://localhost:8080/api/v1/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email: 'user@test.com', password: 'pass123' })
});
const { data: { token } } = await loginRes.c();

// Hacer picks
const picksRes = await fetch('http://localhost:8080/api/v1/tournaments/1/sessions/picks', {
  method: 'POST',
  headers: { 
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    session_id: 1,
    selection_ids: [1, 2, 3]
  })
});
const result = await picksRes.c();
console.log(result.message);
\end{lstlisting}
